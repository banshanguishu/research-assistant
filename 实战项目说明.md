# 实战项目说明

# 从零Vibe Coding 一个AI Agent

## 一、 项目目标

本项目的核心目标是**去工程化、回归原理**。脱离 LangChain、LlamaIndex 等高度封装的框架，直接运用 LLM 的原生 API（如 Function Calling）设计并实现一个具有实际生产力的 AI Agent。

通过手动实现 Agent 的核心循环，你将深入理解 LLM 如何在闭环系统中进行自我纠错、任务拆解以及与外部环境（工具端）的动态交互。这不仅是代码的编写，更是对 **"模型如何思考"** 这一逻辑链条的深度拆解。

## 二、 核心限制（重要！）

为了确保参与者能触及底层逻辑，项目设定以下硬性限制：

1.  **禁止使用Agent框架**：严禁使用 LangChain, LlamaIndex, CrewAI, Haystack 等任何 Agent 编排框架。
    
2.  **语言选择**：Python (推荐版本 3.10+) 或 TypeScript (Node.js 环境)。
    
3.  **调用方式**：LLM调用只能使用OpenAI的 SDK，或者一些底层接口归一化封装来做，比如Pi-ai(TS)。会给大家一套OpenAI Compatible的配置参数
    
4.  **底层循环实现**：必须手动编写并维护 Agent 的 `Thought (思考) -> Action (行动) -> Observation (观察) -> Reflection (反思)` 闭环逻辑。你需要自行管理 `messages` 对话历史数组，并手动解析模型输出的函数调用指令。
    
5.  **完整Review：**整个项目使用VibeCoding的方式进行，但是请搞清楚AI写的每一行代码。**验收的时候随机提问。**
    

## 三、 项目选题

### 深度研究助手 (Research Assistant)

*   **核心功能**：针对用户给出的复杂或模糊话题（例如“分析 2025 年低空经济的商业落地现状”），Agent 需自动进行多维度搜索、信源交叉验证、信息提取并生成一份具备深度见解的报告。
    
*   **技术细节与预期行为**：
    
    *   **Tool Use**：集成搜索引擎 API（如 Tavily, Serper 或 Google Search）。Agent 必须能根据搜索结果提取网页核心文本。
        
    *   **Planning**：Agent 接收任务后，禁止直接搜索全名，必须先产出“研究大纲”或“关键词列表”，分阶段执行搜索（例如：政策背景 -> 核心玩家 -> 市场痛点）。
        
    *   **Reflection (深度验证)**：当搜索到的信息存在冲突或过于笼统时，Agent 需展现出“怀疑精神”，主动发起补充搜索以核实细节，并在报告中注明信源。
        
*   **产出**：一份包含引言、多维度分析、结论及参考文献的结构化 Markdown 研究报告。

    

## 四、 技术要求与设计模式

### 1. 核心模式的深度实现

项目必须在代码逻辑中清晰体现以下模式的融合：

*   **Planning (任务规划)**：实现一种“预计划”机制。Agent 在执行前应输出其执行策略，并在过程中根据 `Observation` 动态调整剩余步骤。这要求你设计精密的系统提示词（System Prompt）来引导模型。
    
*   **Reflection & Self-Correction (反思与自愈)**：
    
    *   **研究助手**：在整合报告前，增加一步“自检”逻辑，检查是否有关键论点缺乏数据支持。
        
    *   **数据处理**：在代码报错时，不仅要传输错误信息，还要传输“错误发生的上下文数据样本”，让模型具备更强的 Debug 成功率。
        
*   **Manual Tool Invocation (手动工具调度)**：你需要解析 LLM 返回的 JSON 字符串（或 Function Call 结构），将其映射到你本地定义的函数，并将函数返回的 `Observation` 封装回 `messages` 数组发送给 LLM。
    

### 2. 代码架构建议

```plaintext
project/
├── src/
│   ├── main.py / index.ts       # 程序启动器，负责加载配置与初始化用户交互
│   ├── engine.py / executor.ts  # Agent 核心状态机。包含 While 循环、Token 计数、异常退出逻辑
│   ├── prompts.py               # 定义 System Prompt，包含对 Thought/Action 格式的严格约束
│   ├── memory_manager.py        # 手动管理上下文窗口，处理长对话时的消息裁剪或总结
│   └── tools/                   # 具体的工具类
│       ├── search_tool.py       # 封装搜索 API 调用及网页解析
│       └── code_sandbox.py      # 安全的本地代码执行环境
├── output/                      # 存放 Agent 生成的临时代码、图表及最终报告
├── .env.example                 # 环境变量模板，注明需要的各种 API Key
└── README.md                    # 重点阐述你的 Agent 循环逻辑是如何设计的

```
